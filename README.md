느낀점 

노성현 : 2학기에 배운 내용이 모두 git과 관련되어있는 내용이였고 그 내용이 조금 어렵게 느껴졌었는데 이렇게 조원들과 함께하여 홈페이지를 만드니까 git에 대해 
        쉽게 생각할 수 있게 되었다. 또한  각자가 메인에서 브랜치를 나누고 맡은 부분을 완성하여 마지막에 메인으로 병합할 때, 오픈소스의 핵심이 협업인 이유를
        알게되었다. 혼자서라면 이 많은 양의 자료와 사진, 브랜치, 코드를 짜지 못 하였을텐데 조원들이 있어서 각자 맡은 역할을 하여 깃 허브라는 매개체를 바탕으로
        프로젝트를 완료했던 것 같다.

김성은 : 이번 프로젝트를 하면서 git을 그냥 수업에서만 배웠을 때랑 다르게, 실제로 써보니까 왜 필요한지 조금은 감이 잡힌 것 같다. 처음에는 파일이 너무 많아서 뭐가 뭔지 헷갈리고, 이미지 올리고 html 수정하는 것도 익숙하지 않아서 시간이 꽤 걸렸던 것 같다. 그래도 계속 하다 보니까 브랜치를 나눠서 작업하는 법이나, 커밋하는 방식 같은 게 자연스럽게 익혀졌다. 또한 사진 넣고 설명 쓰고 레이아웃 맞추는 과정에서 실수도 많았지만, 그걸 git으로 관리하면서 하나씩 고쳐가는 경험이 도움이 됐다. 작업하다가 충돌 나면 당황하기도 했지만 해결 방법을 팀원이랑 같이 찾으면서 협업하는 느낌도 조금 들었다. 무엇보다도 혼자 하면 금방 복잡해질 것 같은 작업들을 git으로 나눠서 관리하니까 덜 헷갈리고, 수정 기록이 남아서 다시 되돌릴 수 있다는 점이 편했다. 이번에 해본 경험이 다음에 다른 과제나 프로젝트 할 때도 도움이 될 것 같고, git을 어렵게만 생각했는데 막상 사용해보니 생각보다 쓸 만하다는 걸 알게 되었다.

김고은 : 이론적으로 학습했던 깃과 브랜치에 대해 직접 만들어보며 배운것을 확실하게 이해할 수 있었습니다. 
홈페이지를 만드는 과정에서 푸쉬, 풀, 커밋 등을 어떻게 하는지를 익혔고 실행하는법까지 학습할 수 있었습니다. 
모둠원들이 각자 맡은 브랜치를 완성시킨 후 마지막으로 합병을 진행하며 협업할 수 있었습니다. 
서로 다른 브랜치에서 작업하니 실수가 나와도 일부만 다시 작업하면 되기 때문에 위험성이 적어 브랜치를 나누는 것이 중요하다는 것을 느꼈습니다.
