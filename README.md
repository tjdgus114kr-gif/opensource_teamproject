# 학교 앞 핫플레이스 추천 & 리뷰 웹사이트

한양대 에리카 캠퍼스 주변의 숨겨진 맛집, 술집, 카페를 학우들 및 교수님들에게 소개하는 웹사이트 프로젝트입니다.사용자에게 친근한 네이버 블로그 형태의 UI를 참고하여 제작하였으며, 실제 학생들이 방문하기 좋은 장소들을 엄선하여 리뷰를 제공합니다.

---

## 1. 팀원 및 역할
| 이름 | 담당 역할 | 상세 내용 |

| **노성현** | **초기 구조 및 카페** | 웹사이트 초기 HTML 골격(Index) 설계, 카페(Cafe) 카테고리 페이지 및 상세 리뷰 구현 |

| **김성은** | **구조 확장 및 술집** | 서브 페이지(Index2) 구조화 및 연결, 술집(Pub) 카테고리 페이지 및 상세 리뷰 구현 |

| **김고은** | **디자인 및 맛집** | 전체적인 CSS 스타일링 및 디자인 통일, 맛집(Food) 카테고리 페이지 및 상세 리뷰 구현 |

## 3. 프로젝트 구조화 및 효과 설명

* **인덱스** : 헤더에 배경 이미지, 왼쪽에 카테고리(맛집,술집,카페) / 가운데 : 인기글 리뷰 1개 표시, 인기글 사진에 마우스를 올리면 그 scale이 0.3초 동안 1.05배 커지는 줌인 효과를 적용, 헤더 이미지 위에 글자가 있는데 이는 header와 header h1의 position을 각각 relative와 absolute로 하여 글자가 위에 있도록 레이어 설정, "학교 앞 핫플레이스" 글자 뒤에 text-shadow를 통해 약한 그림자 적용.
  
* **인덱스2** : 카페,맛집,술집에 각각 존재 헤더, 카테고리는 인덱스와 같이 고정, grid-template-columns: 1fr 1fr 를 사용하여 총 4칸의 그리드 정렬
  
* **상세페이지** : 각 페이지에는 최소 사진 1개와 위치, 대표 메뉴, 리뷰 글 작성. 술집 페이지는 iframe 태그를 사용해 네이버 지도의 기능을 임베드하여 사이트 안에서 사용할 수 있도록 구현.
  
* **전체 페이지** : 모든페이지에는 카테고리와 홈으로 가기가 있어야 함. 카테고리 메뉴에 hover효과 사용으로 마우스를 올렸을 때, 검은 글자가 굵은 파란 글자로 변경, box-shadow를 사용해 가장 우측과 좌측 부분에 그림자 설정, 헤더 이미지를 object-fit: cover를 통해 이미지가 비율이 맞도록 보이게 함. 하냥이의 jpg는 사각형이지만 border-radius을 50%으로 설정하여 원으로 하고 overflow:hidden 을 통해 원 밖으로 나가는 이미지는 자르게 함.

## 4. 느낀점

### 노성현
2학기에 배운 내용이 모두 git과 관련되어있는 내용이였고 그 내용이 조금 어렵게 느껴졌었는데 이렇게 조원들과 함께하여 홈페이지를 만드니까 git에 대해 쉽게 생각할 수 있게 되었다. 또한 각자가 메인에서 브랜치를 나누고 맡은 부분을 완성하여 마지막에 메인으로 병합할 때, 오픈소스의 핵심이 협업인 이유를 알게되었다. 혼자서라면 이 많은 양의 자료와 사진, 브랜치, 코드를 짜지 못 하였을텐데 조원들이 있어서 각자 맡은 역할을 하여 깃 허브라는 매개체를 바탕으로 프로젝트를 완료했던 것 같다. 맨 프로젝트 처음에 조장을 정하기 애매해 혼자 나서서 첫번째 인덱스의 구조랑 코드를 내가 다 짜겠다고 조원들에게 말하였었는데 사실은 좀 막막했었다. 이 프로젝트를 집에서 진행하려고 할 때 개발자였던 형이 도와주려고 하였으나 코드를 이해하지 못하여 깃허브의 매우 기초적인 것만 배우고 다시 혼자서 12,13주차 pdf를 계속 보며 쓰다가 막히는 부분이 있으면 생성형 인공지능을 사용하여 조언을 듣고 그 부분을 해소하였다. 이런 과정에서 언제 태그를 어떻게 사용하는지, 사진의 사이즈는 어떻게 조절하는지, 클래스는 언제 필요한지, 색깔을 정할 수 있는 #??? 태그엔 뭐가 있는지, 특수효과를 사용하는 태그는 무엇이 있는지 등등등 많은 부분에서 놀라울 정도로 이해가 빠르게 되었다. 그 덕에 인덱스를 최대한 빠르게 만들 수 있었고, 그 뒤로는 조원들이 branch를 활용해 짜임새 있는 인덱스의 구조를 사용하여 프로젝트가 매우 매끄럽게 진행되었다고 판단된다. 또한 조원이 만든 index2의 구조를 보고 이것의 코드를 복사하여 그대로 쓴다는 것을 통해 오픈소스의 방식이 소프트웨어의 발전을 가속화 했다는 것을 깨달았다. 해당 조 활동을 통해 내가 만들지 않아도 남이 만든 배포를 허용한 코드를 사용하여 프로그램이나 앱을 만든다면 그 기간을 정말 혁신적으로 줄일 수 있다는 것을 배운 프로젝트였다. 

### 김성은
이번 프로젝트를 하면서 git을 그냥 수업에서만 배웠을 때랑 다르게, 실제로 써보니까 왜 필요한지 조금은 감이 잡힌 것 같다. 처음에는 파일이 너무 많아서 뭐가 뭔지 헷갈리고, 이미지 올리고 html 수정하는 것도 익숙하지 않아서 시간이 꽤 걸렸던 것 같다. 그래도 계속 하다 보니까 브랜치를 나눠서 작업하는 법이나, 커밋하는 방식 같은 게 자연스럽게 익혀졌다. 또한 사진 넣고 설명 쓰고 레이아웃 맞추는 과정에서 실수도 많았지만, 그걸 git으로 관리하면서 하나씩 고쳐가는 경험이 도움이 됐다. 작업하다가 충돌 나면 당황하기도 했지만 해결 방법을 팀원이랑 같이 찾으면서 협업하는 느낌도 조금 들었다. 무엇보다도 혼자 하면 금방 복잡해질 것 같은 작업들을 git으로 나눠서 관리하니까 덜 헷갈리고, 수정 기록이 남아서 다시 되돌릴 수 있다는 점이 편했다. 이번에 해본 경험이 다음에 다른 과제나 프로젝트 할 때도 도움이 될 것 같고, git을 어렵게만 생각했는데 막상 사용해보니 생각보다 쓸 만하다는 걸 알게 되었다.

### 김고은
이론적으로 학습했던 깃과 브랜치에 대해 직접 만들어보며 배운것을 확실하게 이해할 수 있었습니다. 홈페이지를 만드는 과정에서 푸쉬, 풀, 커밋 등을 어떻게 하는지를 익혔고 실행하는법까지 학습할 수 있었습니다. 모둠원들이 각자 맡은 브랜치를 완성시킨 후 마지막으로 합병을 진행하며 협업할 수 있었습니다. 서로 다른 브랜치에서 작업하니 실수가 나와도 일부만 다시 작업하면 되기 때문에 위험성이 적어 브랜치를 나누는 것이 중요하다는 것을 느꼈습니다.
