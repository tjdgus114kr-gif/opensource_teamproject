# 학교 앞 핫플레이스 추천 & 리뷰 웹사이트

한양대 에리카 캠퍼스 주변의 숨겨진 맛집, 술집, 카페를 학우들 및 교수님들에게 소개하는 웹사이트 프로젝트입니다.사용자에게 친근한 네이버 블로그 형태의 UI를 참고하여 제작하였으며, 실제 학생들이 방문하기 좋은 장소들을 엄선하여 리뷰를 제공합니다.

---

## 1. 팀원 및 역할
| 이름 | 담당 역할 | 상세 내용 |
| **노성현** | **초기 구조 및 카페** | 웹사이트 초기 HTML 골격(Index) 설계, 카페(Cafe) 카테고리 페이지 및 상세 리뷰 구현 |
| **김성은** | **구조 확장 및 술집** | 서브 페이지(Index2) 구조화 및 연결, 술집(Pub) 카테고리 페이지 및 상세 리뷰 구현 |
| **김고은** | **디자인 및 맛집** | 전체적인 CSS 스타일링 및 디자인 통일, 맛집(Food) 카테고리 페이지 및 상세 리뷰 구현 |

| **~12/05** | **구조 설계** | Index 페이지 레이아웃 잡기, 카테고리별 기초 블로그 구조 작성 |  
| **12/08** | **디자인 통합** | 팀 회의: 전체 레이아웃 통일, 이미지 배치 및 CSS 디자인 표준화 |

## 3. 프로젝트 구조화

* **인덱스** : 헤더에 배경 이미지, 왼쪽에 카테고리(맛집,술집,카페) / 가운데 : 인기글 리뷰 1개 표시
* **인덱스2** : 카페,맛집,술집에 각각 존재 헤더, 카테고리는 인덱스와 같이 고정, 가운데를 4칸으로 나누어 모든 가게들이 한 눈에 보이도록 표현.
* **상세페이지** : 각 페이지에는 최소 사진 1개와 위치, 대표 메뉴, 리뷰 글 작성. 술집 페이지는 iframe을 활용하여 네이버 지도를 사이트 안에서 사용할 수 있도록 구현.
* **모든 페이지** : 모든페이지에는 카테고리와 홈으로 가기가 있어야 함. 카테고리 메뉴에 hover효과 사용으로 마우스를 올렸을 때 특수효과 발생.

## 5. 느낀점

### 노성현
2학기에 배운 내용이 모두 git과 관련되어있는 내용이였고 그 내용이 조금 어렵게 느껴졌었는데 이렇게 조원들과 함께하여 홈페이지를 만드니까 git에 대해 쉽게 생각할 수 있게 되었다. 또한 각자가 메인에서 브랜치를 나누고 맡은 부분을 완성하여 마지막에 메인으로 병합할 때, 오픈소스의 핵심이 협업인 이유를 알게되었다. 혼자서라면 이 많은 양의 자료와 사진, 브랜치, 코드를 짜지 못 하였을텐데 조원들이 있어서 각자 맡은 역할을 하여 깃 허브라는 매개체를 바탕으로 프로젝트를 완료했던 것 같다.

### 김성은
이번 프로젝트를 하면서 git을 그냥 수업에서만 배웠을 때랑 다르게, 실제로 써보니까 왜 필요한지 조금은 감이 잡힌 것 같다. 처음에는 파일이 너무 많아서 뭐가 뭔지 헷갈리고, 이미지 올리고 html 수정하는 것도 익숙하지 않아서 시간이 꽤 걸렸던 것 같다. 그래도 계속 하다 보니까 브랜치를 나눠서 작업하는 법이나, 커밋하는 방식 같은 게 자연스럽게 익혀졌다. 또한 사진 넣고 설명 쓰고 레이아웃 맞추는 과정에서 실수도 많았지만, 그걸 git으로 관리하면서 하나씩 고쳐가는 경험이 도움이 됐다. 작업하다가 충돌 나면 당황하기도 했지만 해결 방법을 팀원이랑 같이 찾으면서 협업하는 느낌도 조금 들었다. 무엇보다도 혼자 하면 금방 복잡해질 것 같은 작업들을 git으로 나눠서 관리하니까 덜 헷갈리고, 수정 기록이 남아서 다시 되돌릴 수 있다는 점이 편했다. 이번에 해본 경험이 다음에 다른 과제나 프로젝트 할 때도 도움이 될 것 같고, git을 어렵게만 생각했는데 막상 사용해보니 생각보다 쓸 만하다는 걸 알게 되었다.

### 김고은
이론적으로 학습했던 깃과 브랜치에 대해 직접 만들어보며 배운것을 확실하게 이해할 수 있었습니다. 홈페이지를 만드는 과정에서 푸쉬, 풀, 커밋 등을 어떻게 하는지를 익혔고 실행하는법까지 학습할 수 있었습니다. 모둠원들이 각자 맡은 브랜치를 완성시킨 후 마지막으로 합병을 진행하며 협업할 수 있었습니다. 서로 다른 브랜치에서 작업하니 실수가 나와도 일부만 다시 작업하면 되기 때문에 위험성이 적어 브랜치를 나누는 것이 중요하다는 것을 느꼈습니다.
